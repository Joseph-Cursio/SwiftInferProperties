# v1.141 Calibration Plan — Minimal-Counterexample & Replay-Corpus Phase (shrink → auto-regression → replay corpus)

Opens the **minimal-counterexample & replay-corpus epic** (cycles 152–155). First
cross-repo epic: pairs a SwiftPropertyLaws `M5` prerequisite with four SwiftInfer
cycles and one opt-in SwiftIdempotency add-on.

Builds directly on the verify pipeline (v1.42–v1.51) and `Tier.verified` (V1.65) /
`Signal.verifyBothPass`+`verifyDisproven` (V1.66): **verify already runs the
property and moves the tier.** This epic makes the discovered counterexample
*minimal* (shrinking), *durable* (auto-emitted regression in the adopter's own
idiom), and *replayable* (seed + environment corpus, re-checked first on every run).

**Ephemeral.**

## Why now — the three residual holes in the verify loop

| Hole | Current reality (v1.140) | This epic |
|---|---|---|
| No shrinking anywhere | `SwiftPropertyBasedBackend.check()` returns `failingInput` verbatim ("Shrinking is NOT YET exercised (M5+ scope)"). Verify stubs don't even call the backend — they emit hand-rolled `for trial in 0..<100 { … exit(1) }` loops printing the *first* failing value via `VERIFY_DEFAULT_INPUT`. | M5 + V1.141.x |
| Broken seam verify → regression | `verify` prints the counterexample; `convert-counterexample` is a *separate* command needing a hand-typed `--counterexample`. The failure is found, then dropped. | V1.142.x |
| No replay corpus | `VerifyEvidence` stores the *outcome* but not the seed or counterexample. SwiftPropertyLaws' `Seed` (base64 Xoshiro) + `Environment` fingerprint + `ReplayEnvironmentMismatch` guard are entirely unused by SwiftInfer. | V1.143.x |

## Decisions

- **D1 — shrinking lands in SwiftPropertyLaws `M5`, not in stub emitters.** `swift-property-based`'s `Generator<Value, ShrinkSequence>` already carries `_shrinker`; the backend just ignores it. Wiring it once in the kit benefits every PropertyLawKit consumer (now *two*: SwiftInfer verify stubs **and** SwiftIdempotencyPropertyBased — see W6) and avoids N copies of shrink logic across `RoundTripStubEmitter` + siblings. Alternative (inline shrink loop per stub) rejected unless M5 slips.
- **D2 — `--emit-regression` defaults ON for single `--suggestion` runs, OFF for `--all-from-index` survey runs** (avoids flooding `Tests/Generated/`). Overridable both ways.
- **D3 — SwiftIdempotency gets property-based assertions via a new opt-in product**, mirroring the `SwiftIdempotencyFluent` pattern, *not* by adding deps to its lean core. Detailed in the companion design note (see W6).

## Prerequisite — SwiftPropertyLaws `M5` (backend shrinking)

Files: `Sources/PropertyLawKit/Internal/Backends/SwiftPropertyBasedBackend.swift`, `Sources/PropertyLawKit/Public/PropertyBackend.swift`, `Sources/PropertyLawKit/Model/CheckResult.swift`.

- **M5.A** — After the first `.failed`, greedily walk `generator._shrinker(failingInput)`, re-run `property` on each candidate, keep the first still-failing one, repeat to fixpoint (iteration cap ~1000). Requires threading the `Generator` (with shrinker) into `check` — today it takes a bare `sample:` closure, so the `PropertyBackend` protocol gains a shrink-aware path. **The one breaking change in the kit; version accordingly.**
- **M5.B** — Extend the failure payload: `BackendCheckResult.failed(…, shrunkInput:, shrinkSteps:)`; surface on `CheckResult.Outcome.failed(counterexample:shrunkFrom:shrinkSteps:)` (`counterexample` = the minimal string; `shrunkFrom` = the original).
- **M5.C** — Tag a release; SwiftInfer bumps its `SwiftPropertyLaws` pin (`from: "2.5.0"`).
- **Risk R3 (float)** — shrinking `Double` toward `0.0` can cross the `isApproximatelyEqual` tolerance and report a *different* failure than the original; the shrink predicate must reuse the property's exact comparison, not generic `==`.

Tests (match `EffectSymbolTableLawsTests` fixpoint style): shrink output still fails; shrink output ≤ input in shrink order; shrinking terminates; re-shrinking a minimal value is a fixpoint.

## What v1.141 ships

1. **V1.141.A** — Emitters (`RoundTripStubEmitter` + `IdempotenceStubEmitter` / `CommutativityStubEmitter` / `AssociativityStubEmitter` / `StrategistDispatchEmitter`) produce **carrier-native shrinkable `Generator`s** via `DerivationStrategist`, replacing the `Gen<Int>.int(in: 0..<1).map { … }` throwaway pattern (whose shrinker is over a useless `Int`). This is the dependency-heavy core of the epic.
2. **V1.141.B** — Stubs call the kit's now-shrinking backend (`check()` / `checkXxxPropertyLaws`) instead of the hand-rolled `for trial` loop, OR keep the loop but invoke the backend shrinker after first failure. Stubs print `VERIFY_DEFAULT_SHRUNK` + `VERIFY_SHRINK_STEPS` alongside `VERIFY_DEFAULT_INPUT`.
3. **V1.141.C** — `VerifyResultParser` captures shrunk value + step count into `VerifyOutcome`; `VerifyResultRenderer` shows `shrank 14 steps: Complex(8.3e5, -2.1e5) → Complex(0.0, 0.0)`.
4. **V1.141.R2 (graceful degrade)** — when `DerivationStrategist` can't supply a shrinker for a user type, fall back to today's behavior (report first failure, `shrinkSteps: 0`, renderer note) rather than erroring.

## Downstream cycles (summary — own plans at activation)

- **v1.142 — auto-bridge.** On `measuredDefaultFails`, the verify pipeline auto-invokes `ConvertCounterexampleEngine` with the *shrunk* counterexample → `Tests/Generated/SwiftInfer/<template>/<callee>_regression_<hash>.swift`. New `--emit-regression` (D2). **For `idempotence` / `idempotence-lifted` templates, emit the adopter's idiom — a `#assertIdempotent` test — when the target package depends on SwiftIdempotency** (detect via its `Package.swift`); else the generic stub. `VerifyEvidence` → schema v2: add `seed: String?` (base64), `counterexample: String?`, `shrunkCounterexample: String?`, `regressionTestPath: String?` with a v1→v2 migration via the existing `upserting` pattern.
- **v1.143 — replay corpus.** New `Sources/SwiftInferCore/VerifyCorpus.swift` at `.swiftinfer/verify-corpus.json`; entry `{ identityHash, template, seed, counterexample, environment, capturedAt, swiftInferVersion }` reusing SwiftPropertyLaws' `Seed` + `Environment` verbatim. `verify` runs **corpus-first** (inject each stored `Seed` into the stub's `Xoshiro` via the existing `seedHex: SeedHex`, guarded by `Environment` match — skip-with-note on `ReplayEnvironmentMismatch`, never fail), then fresh trials. `verify --replay-only` = CI regression gate. A corpus entry that now *passes* → `drift`-style "property behavior changed" note, not a silent drop.
- **v1.144 — reporting + drift.** One failure block: tier transition, shrink trace, base64 seed (copy-pasteable for manual replay), regression path, corpus size + replay status. `drift` surfaces new corpus entries since baseline.

## SwiftIdempotency tie-in (W6 — companion doc)

SwiftIdempotency is the **adopter-facing** layer: `#assertIdempotent` / `@IdempotencyTests` run `f` twice on *one fixed value* (`__idempotencyAssertRunTwice`) — no generation, no shrinking, no replay. The durable artifact a user keeps lives here, not in `Tests/Generated/SwiftInfer/`. This epic:

1. Makes v1.142's idempotence regression emit a **`#assertIdempotent`** test (the user's idiom), seeded with the shrunk counterexample.
2. Adds **W6 — `SwiftIdempotencyPropertyBased`** (SwiftIdempotency v0.4.0), an opt-in product giving property-based, shrinking, replay-corpus-backed idempotency assertions, reusing M5 shrinking + the v1.143 corpus types. Depends only on M5 + v1.143 corpus → can proceed in parallel with v1.142–v1.144.

Full design: `swiftIdempotency/docs/property-based-assertions-design.md`.

## Adjacent gap (fast-follow, NOT this epic)

RefactorBridge emits kit-protocol conformances (`Semigroup`, etc.) but **does not emit `@Idempotent` / `@ExternallyIdempotent`** when an idempotence suggestion verifies — so a `verified` idempotence property never becomes a durable marker that SwiftEffectInference / SwiftProjectLint would then enforce. "verified ⇒ emit the marker macro" connects the inference loop to the lint loop. Track separately.

## Scope boundaries

- **In scope**: shrinking (kit), shrinkable carrier generators, verify→regression auto-bridge, replay corpus, opt-in SwiftIdempotency PBT product, unified reporting.
- **Deferred**: stateful / model-based testing beyond the W6.C effect-sequence seed (that is epic #2); coverage/classification of generator distribution (`classify`/`collect`); RefactorBridge marker emission (fast-follow above); non-shrinkable carrier shrinker synthesis beyond what `DerivationStrategist` already covers.
