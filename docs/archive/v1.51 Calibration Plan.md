## Risks

1. **Cycle-48 measurement could surface its own asymmetry**. v1.51.A normalizes `Complex` â†’ `Complex<Double>` and the strategist handles Double natively, so 20+2 picks should reach `swift build`. But the swift-numerics `Complex.exp(_:)` function probably depends on a `RealType: ElementaryFunctions` conformance the verifier workdir doesn't import. **Mitigation**: V1.51.F findings doc records build failures explicitly; V1.51.D's E2E test uses a weak assertion (any outcome other than `.architecturalCoveragePending`) so build-time failures still pass the test while surfacing in cycle-48 findings.

2. **Dual-style-consistency curated-pair-list expansion may include incorrect mappings.** The cycle-27 surface output names the non-mutating half (e.g. `formUnion(_:)`); the mutating spelling is inferred. If the cycle-27 picks use a different mutating name than what V1.51.B's table maps to, those picks still fail. **Mitigation**: cycle-48 measurement records which expanded pairs surface and which still fail; v1.52+ refines based on evidence.

3. **The "(b) flip-the-gate" routing change for v1.48 templates may have unintended consequences for the v1.49 hardcoded-path tests**. The current routing assumes v1.46 carriers route to hardcoded path; v1.51.C changes this for v1.48 templates only. Existing tests using `Complex<Double>` carrier with v1.46 templates (round-trip, idempotence, commutativity, associativity) stay on the hardcoded path. **Mitigation**: V1.51.C's unit tests in V1.51.E.3 pin the routing matrix explicitly; existing v1.49.* tests act as the regression suite.

4. **The always-on E2E test could become flaky under Â§13 perf contention.** v1.49.D's `.subprocess` tag mitigates this for the suite-wide cost, but a single E2E test added to the default suite increases the parallel-subprocess count to 22 (was 21 in v1.49). Risk is small (cycle-46 hit 18 cleanly; cycle-46â†’48 trajectory has been stable), but worth noting. **Mitigation**: the new test inherits the `.subprocess` tag; the `--skip VerifyPipelineIntegrationTests` CI pattern still excludes it.

5. **Cycle-48's projected 33-40% measured-fraction assumes v1.51.A-C work as designed.** If any of the three fixes break in unexpected ways, the measurement could land at 22% (just the dual-style picks if A and C fail) or 18% (just the carrier-normalized picks if B and C fail). **Mitigation**: V1.51.E unit tests catch the individual workstreams' regressions; V1.51.F findings doc reports the actual landing honestly.
