## What v1.61 ships

Three workstreams + closeout.

1. **V1.61.A — Fix `DualStyleConsistencyPairResolver` curated pairs**. V1.51.B's SetAlgebra entries treat `formUnion` as both nonMutating and mutating halves (wrong). Per Swift's `SetAlgebra` protocol: `union(_:)` (non-mut) ↔ `formUnion(_:)` (mut), `intersection` ↔ `formIntersection`, `symmetricDifference` ↔ `formSymmetricDifference`, `subtracting` ↔ `subtract`. V1.61.A corrects all 4 pairs.

   Cycle-27 captures the **mutating** name (`formUnion(_:)`) as `primaryFunctionName` (since `formUnion` is more discoverable in source). The resolver's `resolve()` now matches against either field (nonMutating or mutating), with parameter-label stripping for cross-format flexibility. The curated table reduces 9 → 8 entries (the V1.51.B `merge(_:uniquingKeysWith:)` self-pair was redundant with `merging ↔ merge`; removed).

2. **V1.61.B — Mutating-instance-method dual-style emission**. New `composeMutatingDualStylePass` helper, gated on `mutatingInstanceCarriers.contains(recipe.carrierTypeName)` (shares the V1.60.A gate). Emit shape:

   ```swift
   let original = defaultGenerator.run(...)
   let other = defaultGenerator.run(...)
   let nonMutResult = original.<nonMutMethodName>(other)
   var mutCopy = original
   mutCopy.<mutMethodName>(other)
   if nonMutResult != mutCopy { fail }
   ```

   2 values per trial (versus v1.48.B's 1-value, 0-arg shape). Both halves use instance-method call shape with the second value passed as the argument.

3. **V1.61.C — Cycle-58 measurement**. Confirms the 12 dual-style picks reach property check + measure their `.bothPass` / `.defaultFails` outcomes.

4. **Test updates**: V1.51.B's tests pinned the old (wrong) curated pairs. V1.61.A updates them to reflect the correct Swift SetAlgebra mappings. Plus the curated-table-size assertion (9 → 8). All 6 dual-style tests rewritten in V1.51.B's structure but with the right expected values.

5. **V1.61.D-G** bundled closeout — perf baseline, version → 1.61.0, CLAUDE.md, tag + push.
