## Wrapper migration pattern

Introduced for templates whose evidence-line rendering doesn't fit the Constraint Runner's canonical "displayName signature" join. The wrapper preserves bit-for-bit equivalence by:
1. Building the Suggestion via `ConstraintRunner.suggest(constraint:subject:)` — this drives scoring, evidence, identity, carrier.
2. Rebuilding the Suggestion with the template's bespoke `makeExplainability` substituted in.

The template is still "expressed as a Constraint" in the data-driven sense; only the rendering hook is bespoke.
