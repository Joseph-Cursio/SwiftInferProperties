# v1.60 Calibration Plan — mutating-instance-method idempotence emission (first OC pick reaches .bothPass)

Working doc for the v1.60 cut (fifty-seventh calibration cycle). **Tenth Phase 2 gap-closing cycle and first measurable OC closure** — V1.60.A adds mutating-instance-method emit shape for OC idempotence picks; `OrderedSet.sort()` reaches `.bothPass`, the first non-Complex/Double/Int measurement in the project's calibration history.

> **Motivation.** Cycle-56 surfaced 21 `instance-method-shape-not-supported` picks. They split across templates: 5 idempotence on OS (sort + 4 internal-mutating), 12 dual-style-consistency, 4 commutativity/associativity instance methods. v1.60 targets idempotence specifically — small focused scope, smallest emit-shape change, first measurable pick. v1.61+ handles dual-style and commutativity-instance.

**Ephemeral** — archive to `docs/archive/v1.60 Calibration Plan.md` once tagged.

## What v1.60 ships

Two workstreams + closeout.

1. **V1.60.A — Mutating-instance-method idempotence emission.** In `StrategistDispatchEmitter+Templates.swift`'s `composeIdempotencePass`: branch on `mutatingInstanceCarriers.contains(recipe.carrierTypeName)`. For OC carriers (initially just `OrderedSet<Int>`), emit the new shape:

   ```swift
   var onceCopy = value
   onceCopy.<method>()
   var twiceCopy = value
   twiceCopy.<method>()
   twiceCopy.<method>()
   if onceCopy != twiceCopy { fail }
   ```

   Method name extracted by splitting `functionCall` on `.` and taking the trailing component (e.g., `OrderedSet.sort` → `sort`).

   **Side effect on internal-mutating picks**: V1.59.A had matched the swift-frontend CRASH on `OrderedSet._ensureUnique()` etc. as `instance-method-shape-not-supported`. With V1.60.A's new emit shape, the call becomes `copy._ensureUnique()` — Swift now produces a clean "is inaccessible due to 'internal'" diagnostic (no more crash), and V1.56.A's pattern matcher correctly attributes the failure to `internal-api-not-accessible`. Net effect: 4 picks shift `instance-method-shape-not-supported → internal-api-not-accessible` (more accurate category).

2. **V1.60.B — Regression tests**. 2 new tests in `StrategistDispatchEmitterEmitTests`:

   - `idempotenceOrderedSetMutatingShape`: asserts the new emit shape (var onceCopy / twiceCopy + instance method call) for OS carrier.
   - `idempotenceIntCarrierStaticShape`: regression guard — Int carrier still emits the existing static-call shape.

3. **V1.60.E — Cycle-57 measurement.** Confirms `OrderedSet.sort()` reaches `.bothPass`.

4. **V1.60.F — Cycle-57 findings**. Highlights: first non-Complex/Double measured-bothPass; instance-method-shape category drops 21 → 16 (-5; 1 reached .bothPass + 4 reclassified to internal-api).

5. **V1.60.G-I** bundled closeout — perf baseline, version → 1.60.0, CLAUDE.md, tag + push.

## Scope boundaries

- **In scope (v1.60):** V1.60.A idempotence mutating-instance for OS<Int>; V1.60.B regression tests; cycle-57 measurement + findings; standard closeout.
- **Out of scope, deferred to v1.61+:** Dual-style-consistency mutating-instance emission (12 picks); commutativity/associativity instance-method shape (4 picks); strategist recipes for nested-OC + non-OC generics; Comparable-aware monotonicity composer; Phase 2 accept-flow integration.
- **Not changing:** No template additions, no new strategies, no SemanticIndex schema bump.

## Risks

1. **The `mutatingInstanceCarriers` set hardcodes carrier names**. v1.61 expanding to `OrderedDictionary<Int, Int>` etc. requires extending the set. **Mitigation**: small Set<String>; easy to extend; tied to `curatedOCRecipe`'s carriers.

2. **The method-name extraction (`split(.)last`) is brittle**. For function calls like `(+)`-operator or `value.subscript(...)`, the split would misbehave. **Mitigation**: operator-named functions don't have mutating instance forms; subscripts aren't in cycle-27's surface. v1.61+ can refine if cycle-N evidence demands.

3. **The new emit shape assumes the method takes no arguments** (e.g. `sort()`). Other mutating-idempotence picks may have arguments. **Mitigation**: cycle-57 confirms `sort()` works; v1.61+ handles multi-arg mutating methods (e.g. `formUnion(_:)`).
