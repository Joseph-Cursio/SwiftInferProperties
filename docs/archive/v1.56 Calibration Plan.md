# v1.56 Calibration Plan — internal-API build-failure reclassification

Working doc for the v1.56 cut (fifty-third calibration cycle). **Sixth Phase 2 gap-closing cycle and single-workstream release** — closes the last 2 cycle-52 `.measured-error` picks by reclassifying internal-API build failures as `.architectural-coverage-pending` with a specific detail string.

> **Motivation.** Cycle-52 measurement (`docs/calibration-cycle-52-findings.md`) left 2 picks in `.measured-error`: both `Complex.rescaledDivide(_:_:)` (commutativity + associativity). Investigation: the function is declared `internal` in swift-numerics (`Sources/ComplexModule/Complex+AlgebraicField.swift:48`); the workdir's executable target can't access it. The build fails with `error: 'rescaledDivide' is inaccessible due to 'internal' protection level`. Accessibility is a measurement-tooling gap (fix path: skip non-public symbols at indexer time, or `@testable import` in the workdir), not a verifier-architecture gap — so the right category is `.architectural-coverage-pending`, not `.measured-error`.

## What v1.56 ships

Two workstreams + bundled closeout. Smallest cycle since v1.53.

1. **V1.56.A — Reclassify internal-API build failures**. In `VerifyCommand+AllFromIndex.surveyRecord(for:packageRoot:budget:)`: after `runSwiftBuild` returns a non-zero exit code, inspect the combined stdout + stderr for the pattern `"is inaccessible due to '"`. If present, return `.architecturalCoveragePending` with detail `"internal-api-not-accessible"` instead of `.measuredError` with detail `"build-failed: exit=N"`.

   **Why both streams**: cycle-53 measurement debugging showed `swift build` formats compiler diagnostics to stdout (parent-process-readable) when run as a subprocess, not stderr. SwiftPM-level errors (e.g. package-resolution failures) land on stderr. The both-stream check makes the pattern robust against SwiftPM future-version changes.

   **Implementation**: extracted into a testable helper `architecturalPendingDetail(buildStdout:buildStderr:)` returning `String?`. Returns the detail string when the pattern matches, nil otherwise; the caller keeps the v1.52 `.measured-error` classification on nil. Extension point for v1.57+ — additional patterns (e.g. `@_spi` symbols, internal-typed parameters) can be added without touching the call site.

   **Sub-decision — why not skip at indexer time instead**: the cycle-27 fixture index is the v1.29-frozen baseline (109 picks). Skipping non-public symbols at indexer time would retroactively drop the count to 107. The verify-time reclassification preserves the baseline + the picks as evidence — they're now correctly attributed (`.architectural-coverage-pending` with a specific reason) rather than dropped silently. v1.57+ may revisit the indexer-time filter as a methodology improvement.

2. **V1.56.B — Unit tests for the pattern matcher**. 6 tests covering: `internal` / `private` / `fileprivate` access-modifier variations; stdout vs stderr stream detection; nil for unrelated build errors; nil for empty input; matches on either stream. Test count: 2396 → 2402 (+6).

3. **V1.56.E — Cycle-53 calibration measurement**. Re-ran `verify --all-from-index` after V1.56.A landed.

4. **V1.56.F — Cycle-53 findings + roadmap doc**. Standard cycle-N-findings format. §"v1.57+ roadmap" reordered around: (a) TypeShape-driven OC instantiation (60+ picks; dominant remaining category); (b) instance-method emission for chunked-Index picks; (c) methodology guard for binding tables; (d) Phase 2 accept-flow integration (now viable with clean .measured-error = 0 baseline).

## Risks

1. **The pattern check may catch false positives** (a hypothetical user-code error message that includes the same substring). **Mitigation**: the pattern `"is inaccessible due to '"` is Swift-compiler-specific and unambiguous in build-output context; verify-time only sees `swift build` output. False-positive risk is essentially zero.

2. **SwiftPM may change the diagnostic format** in a future version. **Mitigation**: extracted helper makes the pattern easy to update; V1.56.B's unit tests pin the current behavior.

3. **The 2 reclassified picks may belong in `.measured-error` if v1.57 adds `@testable import`** support. **Mitigation**: the reclassification is reversible — re-classify based on whether the workdir uses `@testable`. v1.57+ can refine.
