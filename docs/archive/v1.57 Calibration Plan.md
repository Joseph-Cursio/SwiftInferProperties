Working doc for the v1.57 cut (fifty-fourth calibration cycle). **Seventh Phase 2 gap-closing cycle and methodologically-significant baseline shift** — cycle-27 fixture drops from 109 picks to 103 picks after V1.57.A's scanner-level filter removes 6 file-private declarations that were always noise in the v1.29-era baseline.

## What v1.57 ships

Three workstreams + bundled closeout.

1. **V1.57.A — `private`/`fileprivate` scanner filter**. In `FunctionScannerVisitor.visit(_ FunctionDeclSyntax)`, check the function's modifiers for `private` or `fileprivate`; skip via `.skipChildren` (no append to `summaries`) if present. **Not** filtering `internal` — Swift's default access level is internal, so most user code carries no explicit modifier; filtering internal would over-eagerly drop most picks. Internal-but-explicit symbols (cycle-52's `rescaledDivide`) stay handled at verify time via V1.56.A's pattern matcher.

   **Why not also filter symbols starting with `_`**: that's a convention, not an access modifier. swift-numerics's `_relaxedAdd`/`_relaxedMul` are `public _`-prefixed (kit-internal-by-convention but accessible). Cycle-50/52 confirmed they reach the property check and produce valid `.bothPass` outcomes. Filtering `_`-prefix would drop them — not what we want.

   **Unit test**: synthetic source with 5 functions (public, private, fileprivate, internal, default-internal) → 3 captured, 2 skipped.

2. **V1.57.E — Cycle-27 fixture rebuild + cycle-54 measurement**. After V1.57.A lands, the per-checkout indexes are deleted and rebuilt via `swift-infer index` against each of the 4 cycle-27 corpus checkouts. Merged fixture index drops from 109 to 103 entries. Then `verify --all-from-index` runs against the new index to produce the cycle-54 survey.

   **Per-checkout drop breakdown**:
   - swift-numerics: 20 (unchanged)
   - swift-algorithms: 8 (unchanged)
   - swift-collections: 74 (unchanged)
   - **SwiftPropertyLaws: 7 → 1 (-6 private declarations)**

   The 6 dropped picks (all from SwiftPropertyLaws):
   - 3 file-private helpers in `*CollectionLaws.swift`: `walkCap(for:)`, `iterationCap(for:)`, `snapshot(_:)`
   - 3 `private static` members of `ViolationFormatter`: `headerLine(_:)`, `nearMissLines(_:)`, `formatBuckets(_:)`

## Risks

1. **The fixture-baseline shift invalidates direct cross-cycle comparison**. Cycles 47-53 reference "109 picks"; cycle-54 onward references "103 picks". **Mitigation**: findings doc documents the shift explicitly; cross-cycle comparisons normalize against the 103 baseline going forward; the 6 dropped picks were always noise (private declarations).

2. **Other cycle-27 corpora may have more private declarations than this rebuild caught**. **Mitigation**: cycle-54 rebuild confirms the per-checkout counts. swift-numerics / swift-algorithms / swift-collections kept their counts (20/8/74) — no surprises in those corpora. SwiftPropertyLaws's drop was expected.

3. **A future `swift-infer index` run on a user's codebase will produce a smaller index than pre-V1.57**. Could be surprising. **Mitigation**: documented in `Sources/SwiftInferCore/FunctionScanner.swift`'s V1.57.A comment; v1.57.0 release notes flag the change.

4. **The 6 dropped picks could be re-added if a future cycle reverses the filter**. **Mitigation**: V1.57.A's unit test pins the current behavior; reverting would require deleting the test.
