# v1.38 Calibration Plan — Constraint Engine Batch Migration (#3, #4, #5)

Working doc for the v1.38 cut. v1.38 is the **thirty-fifth calibration cycle** and the **first batch-migration cycle** for the Constraint Engine refactor (PRD §20.2). Migrates three templates in one cycle via the V1.36.D mechanical pattern.

> **Why batch now.** V1.36 (Commutativity) + V1.37 (Monotonicity) validated the pattern on 2 representative unary-summary shapes with different runtime-input cardinalities. V1.37.C confirmed the pattern held without modification. The next 3 simplest templates (Associativity, InvariantPreservation, DualStyleConsistency) span a range of complexity — but all map cleanly to the established pattern. Batching them in one cycle accelerates the refactor without sacrificing the bit-for-bit equivalence guarantee.

**Ephemeral** — archive to `docs/archive/v1.38 Calibration Plan.md` once tagged.

Three migrations + bundled closeout:

1. **V1.38.A — AssociativityTemplate**. Subject: `FunctionSummary`. Runtime inputs: `vocabulary`, `reducerOps`, `inheritedTypesByName`. Caveats: 3-fixed (FP advisory OR fallback warning).
2. **V1.38.B — InvariantPreservationTemplate**. Subject: `FunctionSummary`. Runtime inputs: none (keypath derived from summary). Caveats: 3-constant.
3. **V1.38.C — DualStyleConsistencyTemplate**. Subject: `DualStylePair`. Runtime inputs: `carrierKindResolver`. Caveats: 2-constant.
4. **V1.38.D**: Equivalence tests per template.
5. **V1.38.E–H**: cycle-35 findings + perf + version + closeout + tag (bundled).

After v1.38: **5 / 10 templates migrated**. Remaining 5: round-trip, idempotence non-lifted + lifted, inverse-pair non-lifted + lifted, identity-element non-lifted + lifted, composition. Likely v1.39 + v1.40 to complete.

## Risk

- **Risk**: a template I batch has a subtle quirk that breaks bit-for-bit equivalence (e.g., conditional caveats with side-channel ordering). **Mitigation**: equivalence test per template against ≥5 fixtures + the full pre-migration test suite continuing to pass without modification.
