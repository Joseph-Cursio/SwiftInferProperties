# v1.2 Release Plan

Working doc for the SwiftInferProperties v1.2 cut. With TestLifter M13 (general partition surface for equivalence classes — three of four Option A axes), TestLifter M14 (same-target enum coverage for N-class `coversDomain`), and TestLifter M15 (`Float`/`Double` numerical-bound preconditions, closing the M9 plan OD #1 deferral) all closed on `main`, the §7.8 trio is now functionally complete for the v1.x scanner shape:

- First example (preconditions) covers all four `ParameterizedValue.Kind` cases the M4.1 scanner produces — `Int` / `String` / `Bool` from M9, `Double` from M15.
- Second example (round-trip-pair domain narrowing) shipped at M10.
- Third example (equivalence classes) covers three of four Option A axes via M11 + M13 + M14, with same-target enum exhaustiveness annotation fully wired.

v1.2 is the version cut that captures M13 + M14 + M15. The underlying work is already shipped, so this plan is mechanical (perf re-baseline + version + CHANGELOG + archive). **Ephemeral** — archive to `docs/archive/v1.2 Release Plan.md` once the tag is pushed and the §13 + §16 release-blocking criteria still hold.

> **v1.2 is a feature cut, not a posture cut.** v0.1.0 closed the release-blocking guarantees and tagged the M1–M8 + TestLifter M1 surface; v1.1 added TestLifter M2–M11 (the §7.8 trio's first cut). v1.2 widens the §7.8 trio's third example to the general partition surface (M13 + M14) and closes the §7.8 first example's `Float`/`Double` deferral (M15). The §16 hard guarantees are unchanged; the §13 numbers are re-verified rather than recalibrated.

## What v1.2 ships

The v1.2 line was opened by M13 and closed by M15. Each milestone shipped with its own M-series acceptance bars; v1.2 is the version that surfaces them to consumers.

### TestLifter M13 — General partition surface for equivalence classes

PRD v1.0 §7.8 third example, scope A axes 1+2+4. Lifts `MarkerPair` to `SwiftInferCore.MarkerTable.swift` + adds `MarkerSet` + the combined `MarkerTable` carrier; `Vocabulary.markerPairs` / `markerSets` JSON round-trip. M13.1 broadens the discover-loop scan from `[Valid/Invalid]` to `MarkerTable.curatedPairs` (5 pairs: `Valid`/`Invalid` + `Success`/`Failure` + `Accept`/`Reject` + `Pass`/`Fail` + `Allowed`/`Forbidden`); per-predicate ranking dedup picks the highest-site-count winner when a predicate fires under multiple pairs. M13.2 ships `NClassEquivalenceClassDetector` + `NClassEquivalenceClassHint` for ≥3-bucket partitions on `XCTAssertEqual(predicate(x), .case)` / `#expect(predicate(x) == .case)` shapes; reuses M11 vetoes + adds `PredicateVetoReason.predicateReturnNotEquatable` (textual proxy for the full Equatable check, no SemanticIndex). M13.3 wires both detectors through `LiftedSuggestionPipeline` + `EquivalenceClassHintKind` sum-type side-map + accept-flow renderer (N-class file naming `EquivalenceClasses_<predicate>_<markerSetName>.swift`); pipes `Vocabulary.markerSets` into `TestLifter.discover` as additive marker-table extension. Two-class `coversDomain` annotation fires syntactically (XCTAssertTrue + XCTAssertFalse, no `!` negation) → renderer surfaces `Exhaustiveness: forAll x: T. p(x) ∨ ¬p(x)`.

Sub-milestones M13.0–M13.3 closed in `docs/archive/TestLifter M13 Plan.md`. Multi-predicate equivalence classes (axis 3) explicitly deferred — same SemanticIndex-sequencing constraint as M12.

### TestLifter M14 — Same-target enum coverage for N-class `coversDomain`

PRD v1.0 §7.8 third example, axis 4 N-class branch. M14.0 extends `TypeDecl` with `enumCaseNames: [String]`, populated by `FunctionScannerVisitor.makeTypeDecl` for primary `enum` decls + extensions that add cases; `MemberBlockInspector.enumCaseNames(in:)` walks `EnumCaseDeclSyntax` and strips associated values + raw-value initializers. M14.1 widens `NClassEquivalenceClassDetector.detect(...)` with `typeDecls: [TypeDecl] = []`; `computeCoversDomain` unions same-name primary + extension records, runs case-insensitive identifier coverage, sets `hint.coversDomain == true` only when every same-target enum case is matched by a marker (cross-target / unresolved / partial / empty / optional-return / function-typed all conservative-false). M14.2 threads `artifacts.typeDecls` through `LiftedSuggestionPipeline.equivalenceClassLifted` + `equivalenceClassHintMap` from `Discover+Pipeline`; the M13.3 renderer's `Exhaustiveness:` comment now surfaces in production for fully-covered N-class corpora.

Sub-milestones M14.0–M14.2 closed in `docs/archive/TestLifter M14 Plan.md`. Cross-target enum case enumeration explicitly deferred (SemanticIndex territory, sibling to M12 / M13.+).

### TestLifter M15 — `Float`/`Double` numerical-bound preconditions

PRD v1.0 §7.8 first example. Closes the M9 plan OD #1 deferral. M15.0 extends `PreconditionPattern` with `positiveDouble` / `nonNegativeDouble` / `negativeDouble` / `doubleRange(low:high:)`; renderer + accept-flow stay string-driven (per-kind `describePattern` / `suggestedGenerator` helpers in `LiftedTestEmitter+Generators` + `PreconditionInferrer`). M15.1 replaces `case .float: return nil` with `detectFloatPattern` + `parseDoubleLiteral`; underscore-tolerant, explicit `0x`/`0X` rejection (mirrors M9's hex-radix kill — Swift's `Double.init(_:)` natively parses `0x1.0p2` → 4.0), `!isFinite` defensive kill, M9 OD #4 most-specific rule preserved (≥2 distinct → `doubleRange`; else sign-bound). M15.2 ships an end-to-end fixture in `MockInferredPreconditionIntegrationTests` exercising 5 distinct `Doc(title:, ratio:)` Double sites; `// Inferred precondition: ratio — all observed values are in [1.5, 5.5]` + `Gen.double(in: 1.5...5.5)` surface in production output.

Sub-milestones M15.0–M15.2 closed in `docs/archive/TestLifter M15 Plan.md`. After M15, the M9 inferrer covers all four `ParameterizedValue.Kind` cases the M4.1 scanner produces.

## Open decisions

1. **CHANGELOG entry granularity for M13's MarkerTable lift.** The `MarkerTable` carrier shape (M13.0 lift of `MarkerPair` to `SwiftInferCore` + `MarkerSet` addition) is a real public-API surface, not a private impl detail — `Vocabulary.markerPairs` / `markerSets` are persisted in `decisions.json`. Should the CHANGELOG mention this as a separate bullet under "TestLifter" or fold it into the M13 line? Default proposal: **(a) fold into M13 line as a parenthetical** (mirrors v1.1's posture for the §13 row 4 side-map carrier — implementation detail folded into the milestone line). Rationale: the consumer-facing surface is "M13 ships generalized partitions"; the marker-table shape is the implementation that makes it work. The release-plan doc + archive plan record the carrier shape for future contributors.

2. **Perf baseline diff against v1.1.** The v1.2 baseline file should either (a) replace the v1.1 baseline as the canonical regression anchor or (b) sit alongside as `docs/perf-baseline-v1.2.md` with v1.1 retained for forensic comparison. Mirrors the v1.1 cut's posture vs v0.1.0; the v1.1 baseline retains forensic value for understanding the M13/M14/M15 trajectory's perf delta over the same six-detector-fan-out run.

5. **Whether the README should call out that the §7.8 trio is functionally complete.** v1.1 README didn't make this kind of claim; v1.2 closes the trio fully. Default proposal: **(a) keep README posture-neutral** — list features at v1.2 without any "we shipped the trio" framing. ## v1.2 trajectory framing

PRD v1.0 §20 ("v1.1+ trajectory") describes a list of post-v0.1.0 work items: TestLifter M10 + M11 (shipped at v1.1), SemanticIndex, IDE integration, `swift-infer apply`, `swift-infer metrics`. v1.2 closes the §7.8 trio's remaining gaps (M13 generalized partitions, M14 same-target enum coverage, M15 `Float`/`Double` preconditions) but leaves the §20 surface largely open — v1.2 doesn't promise the §20 items, only the §7.8 close-out.

After v1.2 ships, the natural next contributions are:
- **General consumer-producer chain detection** (M10's deferred Option A) — extends §7.8 example 2 beyond round-trip pairs. Not §20 per se; a v1.x narrow follow-up. SemanticIndex-independent.
- **Multi-predicate equivalence classes** (M13's deferred axis 3) — extends §7.8 example 3 beyond predicate-negation buckets. SemanticIndex-blocked (sibling to M12).
- **Cross-target enum coverage** (M14's deferred bit) — extends axis 4 beyond same-target enums. SemanticIndex-blocked.
- **PRD §20 SemanticIndex** — biggest v1.x lift. Was originally deferred from v1.0; the constraint-engine + index unlock the IDE integration + `apply`/`metrics` items downstream and cleanly enables M12 / M13.+ / M14 cross-target.
- **PRD §20 IDE integration** — depends on SemanticIndex.
- **PRD §20 `swift-infer apply`** — moves from "all output is opt-in via `--interactive`" to "single command applies a baseline set of accepted suggestions." Depends on a stable v1.1 baseline-decisions surface (which already exists at v0.1.0).
- **PRD §20 `swift-infer metrics`** — surfaces the `.swiftinfer/decisions.json` aggregation per PRD §17.

No active milestone plan opens with this commit. Subsequent work picks one of the above and opens a fresh `docs/<plan>.md`.
