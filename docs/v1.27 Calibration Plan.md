## The two workstreams

### Workstream A — Algo Iterator-like survivors veto

**What:** Extend V1.21.A's IteratorProtocol carrier veto with a third detection path: **fires when the carrier is a corpus-defined Sequence-conforming type AND the method name is in `iteratorMethodNames`** — even when the carrier doesn't explicitly conform to IteratorProtocol.

V1.21.A's current paths:
1. Primary: textual conformance to `IteratorProtocol` via `inheritedTypesByName`.
2. Name fallback: carrier name `Iterator` or `*Iterator` suffix + curated method name.

V1.27.A adds:
3. Sequence-fallback: carrier conforms to `Sequence` (via `inheritedTypesByName`) AND method name in `iteratorMethodNames`. The Sequence protocol's contract requires its `makeIterator()` to return a stateful iterator; types that define `mutating func next()` / `advance()` directly on a Sequence (rather than via a nested Iterator) follow the same stateful pattern.

**Cycle-23 origin:** 2 Algo picks measured REJECT on idempotence-lifted (cycle-23 #14, #15). The carriers are likely `Chain`-family or similar Sequence-conforming Algo structs that expose `mutating func next()`-style methods directly.

**Implementation site:** modify `IdempotenceTemplate.iteratorProtocolCarrierVeto(for:inheritedTypesByName:)` in `IdempotenceTemplate+IteratorVeto.swift` to add the Sequence-conformance path.

**Cost:** ~1-2 hours. ~6 unit tests.

**Plan-vs-actual projection:** **closes 2 Algo candidates** (the cycle-23 #14 + #15 picks).

### Workstream B — Bucket/word direction-pair inverse-pair veto

**What:** Apply V1.25.A's name-prefix-gated magnitude bump to `InversePairTemplate.directionLabelCounterSignal(for:)`. Currently V1.11.1 fires at -10 (either side direction-labeled, smaller than idempotence's -15 because inverse-pair's baseline is +25 not +30). V1.27.B bumps to **full veto** (`Signal.vetoWeight`) when:

1. **Both** pair sides have direction-labeled first parameters (both-sides match), AND
2. Both pair sides' function names start with `index`, `bucket`, or `word`.

**Cycle-23 origin:** 2 OC inverse-pair picks measured REJECT (cycle-23 #26, #27): `bucket(after:) × bucket(before:)` and `word(after:) × word(before:)`. Both have direction-labeled first params + name-prefix match.

V1.25.A's idempotence parallel: name-prefix + direction-label = full veto. V1.22.B's round-trip parallel: both-sides direction-label = -25. V1.27.B = both-sides + name-prefix on inverse-pair = full veto.

**Implementation site:** modify or extend `InversePairTemplate.directionLabelCounterSignal(for:)` in `InversePairDirectionLabelCounter.swift`.

**Cost:** ~1-2 hours. ~6 unit tests.

**Plan-vs-actual projection:** **closes 2 OC candidates** (the cycle-23 #26 + #27 picks).
