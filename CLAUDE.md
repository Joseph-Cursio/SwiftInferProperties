# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository. **This file is a pointer-only index** — per-cycle narratives live in `git log` + `docs/calibration-cycle-N-findings.md` + `docs/calibration-cycle-N-data/`.

## What this repo is

**SwiftInferProperties** — type-directed property inference for Swift. Surfaces idempotence, round-trip pairs, and algebraic-structure (semigroup / monoid / group / semilattice / ring) candidates from function signatures, cross-function pairs, and existing unit tests. All output is human-reviewed; nothing auto-executes.

One-way downstream of [SwiftPropertyLaws](https://github.com/Joseph-Cursio/SwiftPropertyLaws):

```
SwiftInferProperties → SwiftPropertyLaws (PropertyBackend, DerivationStrategist) → swift-property-based
```

## Design decisions baked in (follow rather than re-litigate)

- **Conservative inference — high precision, low recall.** PRD §3.5. When in doubt, fewer suggestions.
- **Opt-in, human-reviewed output.** Never auto-applies/executes/commits. CI mode emits warnings, not failures.
- **Avoid the Daikon trap.** Too many suggestions → raise thresholds, don't pile on filters.
- **Explainability is a first-class output.** Every suggestion ships "why suggested" + "why this might be wrong." PRD §4.5.
- **Generator inference delegates to SwiftPropertyLaws.** Call `DerivationStrategist`; don't reimplement. PRD §11.
- **Relaxed partial-exploration is allowed for `.tca` interaction verify (cycle 124).** A `measured-bothPass` may be established over the *constructible-action subset* (payload-free + raw cases), skipping non-derivable composition cases (binding / nested `X.Action` / delegate / `Result`) — accepting that a counterexample reachable only via a skipped action is missed, in exchange for ~4× reach. **Binding guardrail:** every partial verdict MUST disclose the excluded set (`verified over M of N action types (excluded: …)`) in its `detail` + render; the witness itself must still be constructible. A partial `bothPass` promotes idempotence `.likely → .verified` (with the annotation); a later full-action-space verification supersedes it. This is the one deliberate carve-out from the strict all-actions-or-reject posture — see `docs/calibration-cycle-124-findings.md`.
- **A measured `bothPass` overrules the Finding-G `.possible` pin (cardinality / biconditional) ONLY at full action-space coverage (cycle 135).** The gated families are pinned because the static smell holds only 33–50% at runtime (often UI-layer-enforced). A *full-coverage* measured bothPass (`excludedCaseNames.isEmpty`) is sound per-candidate proof the reducer enforces the invariant itself → overrule the pin and promote `.possible → .verified` (30 + 50 = 80 → `.strong` → promoted, via a measured-evidence-only carve-out in `InteractionVerifyEvidenceScoring`, NOT a change to `tier(forScore:)`). A **relaxed / partial** bothPass does **not** overrule — it stays `.possible` — because cardinality's failure mode lives in exactly the action types relaxed exploration excludes (`binding` / `PresentationAction` / nested `X.Action`), so a partial bothPass is systematically biased toward false-pass. **Binding guardrail:** a promoted gated verdict MUST disclose that the pin was overruled by full-coverage measured execution (0 excluded). Static score alone still never overrules. Ratified, not yet built — see `docs/calibration-cycle-135-findings.md`.
